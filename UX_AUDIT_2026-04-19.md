Mix of code review + live browser testing as scholar and teacher. Nothing is committed.

## Part 1 — Fixes already applied (uncommitted, safe-to-revert)

Short list of what I changed and why. Each is tiny and low-risk; revert any that are wrong.

1. **AI greeting no longer uses auto-generated usernames** — `convex/projectHelpers.ts`. Added `friendlyScholarName(name, username)` that returns `null` when `name === username` or the name contains digits/underscores (real first names don't). For a freshly-registered scholar who skips profile setup, the AI will now just say "Hi!" instead of "Hello testkai_1776564548270!".

2. **Scholar project card hides teacher-facing `analysisSummary`** — `app/scholar/page.tsx`. The observer's summary ("Brief fact-checking query; no engagement with conceptual understanding") was rendering on the scholar's own home. Gated behind `isRemoteMode` so teachers in remote view still see it, scholars don't. **This is the item you asked me to double-check — pre-fix the render had no role gating, and the Convex query returns the summary to the scholar too. My fix aligns with your "teachers see annotations scholars don't" model. If you actually intended scholars to see this, revert.**

3. **Empty-state message on scholar home** — `app/scholar/page.tsx`. When a scholar has zero projects and zero seeds, the page was totally blank. Added a centered "Ready when you are — tap New Project to start something new."

4. **Branded 404 page** — new file `app/not-found.tsx`. Replaces the default Next.js stark `404 | This page could not be found.` with a branded card matching the auth screen.

5. **`/login` → `/sign-in` redirect** — new file `app/login/page.tsx`. Anyone guessing `/login` now lands on the real sign-in page instead of a 404.

6. **ErrorBoundary no longer dumps raw Convex stack traces** — `components/ErrorBoundary.tsx`. On sign-out the teacher dashboard was showing `[CONVEX Q(users:listScholars)] [Request ID: …] Server Error Uncaught Error: Not authenticated at requireUser (../../convex/lib/auth.ts:25:9)…` — visible file paths and all. Now: (a) "Not authenticated" errors auto-redirect to `/sign-in`, (b) other errors show a generic "Please try again" without the server stack.

7. **AuthForm surfaces specific backend errors** — `components/AuthForm.tsx`. "Invalid invite code" from the server now shows "That invite code isn't valid. Check with whoever invited you." instead of the all-encompassing "Invalid invite code or username already taken".

8. **AuthForm button shows progress** — `components/AuthForm.tsx`. Button text changes to "Creating account…" / "Signing in…" while submitting. Before it just looked unresponsive.

9. **EntityManager delete → archive confirmation** — `components/EntityManager.tsx`. One-click trash-icon archive of units/personas/perspectives/processes now pops a `window.confirm` ("Archive 'X'? Scholars will no longer see it…"). Also renamed aria-label from "Delete" to "Archive" since it's a deactivation, not a hard delete.

10. **ProjectViewer analyze / whisper show feedback** — `components/ProjectViewer.tsx`. Both handlers previously only did `console.error` on failure. Added toaster success/error notifications so teachers know whether their whisper actually saved.

11. **`<start>` system message hidden in teacher recent-messages view** — `components/ScholarProfile.tsx`. The synthetic `<start>` greeting trigger was showing up as a "scholar" message with no content in the activity feed.

12. **Tab-hold walkie-talkie excludes TEXTAREA and contenteditable** — `components/ProjectInterface.tsx`. Previously excluded only INPUT/SELECT, so holding Tab while in the chat textarea was hijacking focus navigation for keyboard users.

13. **Chat placeholder is friendlier** — `components/ProjectInterface.tsx`. `"Chat"` → `"Type a message or question…"`. Focus-mismatch placeholder gained "your" ("your teacher set a different activity").

14. **Chat textarea `maxLength={4000}`** — `components/ProjectInterface.tsx`. Prevents accidental giant pastes from a kid copying a whole Wikipedia page.

Type-checks (`tsc --noEmit` for both `convex/` and root) pass clean.

## Part 2 — Discussion items (not applied — want your input)

### A. Scholar's own "Fading" / "Idle" self-status label

`components/StatusOrb.tsx` labels scores 0–5 as Idle → Fading → Waking Up → Warming Up → Engaged → Locked In → Deep Focus → On Fire → Learning Furiously. The popover shown to scholars on their own avatar shows the current label. "Fading" on a 7-year-old's screen reads as judgment. But this might be intentional — self-awareness, not judgment, and the whimsical scale is part of the brand. **Question:** leave as-is, or soften the low-score labels for scholars only (teachers still see the real scale)? Easiest swap: "Idle" stays, "Fading" → "Settling", "Waking Up" stays.

### B. Teachers see every scholar globally

When `testteacher` (fresh account, invite code `alohahonu`) signed in, the Scholars tab listed **every scholar in the system** — including real enrolled scholars plus my just-made test scholar. Is that intentional for a school-wide directory? Most LMSes scope to classes/assignments. If you ever onboard a second teacher who shouldn't see every family, this becomes a privacy issue.
> Yes , add this to the future work section of the TODO list... not a problem until we scale more

### C. Profile setup is half-required

`app/scholar/page.tsx:396-400` — `ProfileEditModal` opens for first-time scholars with `onClose={() => {}}`. So the backdrop click and Escape do nothing. But the modal has a "Skip for now" button inside that closes it and flips `profileSetupComplete = true`. So it IS skippable, just via the only visible button. Either (a) make it truly required (no Skip button) or (b) make Escape/backdrop work and rename "Skip for now" to something more honest like "Do this later." Also: when a scholar skips, their `name` stays equal to their username, which drives the ugly AI greeting I patched above. Long-term fix is probably: don't auto-copy username into name on `registerWithCode` (`convex/users.ts:54`).
> lets make the name required but photo grade level optional

### D. Name field on profile setup is empty / no placeholder

Inputs on the "Welcome! Set up your profile" dialog have no placeholders. Kids type into a blank Name box with no hint whether it's first name, full name, or a nickname. Low-risk fix: add `placeholder="Your first name or what you like to be called"`. Same for Date of Birth — a line of context ("Helps us match you to the right reading level") would explain why you're asking.
> yes on the DOB, no on the name for now

### E. Scholar home project card title truncation

Project cards use the auto-generated title from the first message. If a scholar types a long first message, the title is their whole sentence truncated with ellipsis. Consider: (1) a server-side summarizer that produces a short phrase title, (2) letting scholars rename projects (they already can in the header, but it's not obvious on the home card).
> yes, make it so

### F. "Welcome back" on a brand-new scholar's first project

`convex/prompts.ts:59` tells the AI to greet warmly, and the AI said "Welcome back" on a first-ever project. That's a model-quality nit, not a bug you can fix in code reliably — but worth noting: the prompt could say "greet them (it might be their first project — don't say 'welcome back')".
> yes, fix the prompt
### G. Curriculum grid shows untitled "New Unit" entries

Two tiles in Curriculum → Units are literally titled "New Unit" with 5 and 9 lessons. Someone created a unit via the dashed "+ New Unit" card and never renamed it. Two fixes to consider: (1) require a title before the unit is saved, (2) don't render un-named units until they have a title.
> yes on # 1
### H. Rabbithole vs Tradewinds Learn branding

The codebase is named `makawulu` and `CLAUDE.md` calls it "Tradewinds Learn", but the UI still says "Rabbithole" everywhere (`Join Rabbithole`, page title, header logo text in `AppLogo`). If the rename is deliberate-not-yet-rolled-out, fine. If it's orphaned, one `find + replace` across `components/AppLogo.tsx`, `app/layout.tsx` (page title), and `AuthForm.tsx` `config.signUp.heading` would do it.
> yes it should be called Rabbithole everywhere. fix 

### I. Sign-out flakiness

From the scholar view: clicking the sidebar "Sign out" icon once did nothing (closed sidebar on the second click). The avatar dropdown "Sign Out" worked. Two entry points, inconsistent behavior. Also the sign-out action sometimes landed me back on `/scholar` instead of `/sign-in` — I had to navigate manually. Worth tracing. Probably `signOut()` is fire-and-forget and the router push races it.
> ok - investigate and fix
### J. Scholar profile Activity tab shows 9 tabs

`components/ScholarProfile.tsx:81-91` renders tabs: Activity / Mastery / Seeds / Standards / Strengths / Documents / Notes / Dossier / Reading & Audio. On a 13" laptop this wraps oddly. Consider grouping: "Activity" + "Notes" under "Now", "Mastery"/"Standards"/"Strengths" under "Progress", "Dossier"/"Seeds"/"Documents"/"Reading" under "Library". Or just accept it.
> let's add this to the TODO as future work

### K. Forced password reset modal has no escape hatch

`components/SetPasswordDialog.tsx` when invoked via `mustResetPassword` has `closeOnInteractOutside={false}` and no visible Cancel. If a scholar hits it but forgot the temp PIN, they're stuck. Consider a "Contact teacher" link.
> hodl off for now, add to teacher work

### L. Teacher whisper UX

`components/ProjectViewer.tsx` — the "Teacher Whisper" field is labelled "Guidance injected into AI system prompt" but doesn't explain when it takes effect (next turn? immediately? persistent?). Add a short helper line.
> Takes effect on next turn

### M. Teacher can see any scholar's account by clicking

In the Scholars list I could click any scholar with no access check visible. Tied to B above — scope may be intentional. Noting for completeness.
> future work like B

### N. Welcome-to-profile modal has "Skip for now" in low contrast

The Skip button is a plain text link in light gray. On mobile especially, kids may miss it and think they're stuck. Either strengthen the contrast or label the primary button "Save" vs "Skip" with equal weight.

### O. Empty-activity message on teacher page is redundant

The teacher "Activities" tab shows both:
- Left panel: "Click Start to begin an activity"
- Main panel: "Start an activity to begin"

Two different strings for the same thing. One is enough.
> true, fix

### P. Password validation inconsistency

`components/AuthForm.tsx:67` enforces 4-char minimum client-side; `convex/auth.ts:9` enforces the same server-side. Both say "Password must be at least 4 characters". Good. But there's no hint on the form — users type a 3-char password and only get told after submit. Adding `helperText` under the password field is a 2-minute win. (I didn't apply because Chakra `Input` doesn't have direct helperText; needs a surrounding `Field` pattern.)
>fix

### Q. Invite code field has no placeholder hint

`components/AuthForm.tsx:185` — placeholder is literally "Invite code", which gives no hint what format to use. Since codes are `alohamoli` / `alohahonu` / `alohakolea`, consider `placeholder="e.g. alohamoli"` — though that might leak real codes. Safer: put the hint on the sign-up link text ("Need an account? Create one with your invite code").
> do nothing
